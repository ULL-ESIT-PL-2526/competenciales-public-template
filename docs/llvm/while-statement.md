## Example 

See [examples/llvm/llvm-20-while.drg](/examples/llvm/llvm-20-while.drg) 
  
```C
{
  int i; 
  while ( i < 10 ) {
      i = i + 1;
      print(i);
  }
}
```

See the generated code at [tmp/llvm-20-while.ll](/tmp/llvm-20-while.ll)

```ll 
; ... target information, global symbols, etc. ...
define i32 @main() {
  %.i.0.addr = alloca i32                   ; int i; %.i.0.addr is generated by const address = ctx.nextAllocaName(dragonName) when visiting VariableDeclarator for i
  store i32 0, i32* %.i.0.addr              ; i = 0; // We do automatic initialization to 0. ctx.emit(`  store ${llvmType} ${coerced.value}, ${llvmType}* ${address}`) when visting VariableDeclarator with init.
  br label %while.cond_0                    ; if (!lastEmittedIsBranchOrReturn(codeBeforeWhile)) { ... }
while.cond_0:                               ; The label was generated with: const condLabel = ctx.nextLabel('while.cond') at enter
  %tmp_a = load i32, i32* %.i.0.addr.       ; Condition code (condCode) emitted during traversal of node.test
  %tmp_b = icmp slt i32 %tmp_a, 10          ; const condCode = path.get('test').getData('emitedCode'); 
  br i1 %tmp_b, label %while.body_1, label %while.end_2 ; We obtain %tmp_b with: const condVal = getNodeValue(nodeValues, node.test);

while.body_1:                               ; const bodyLabel = ctx.nextLabel('while.body') when entering the WhileStatement
  %tmp_c = load i32, i32* %.i.0.addr        ; Body code (bodyCode) emitted during traversal of node.body 
  %tmp_d = add i32 %tmp_c, 1                ; const bodyCode = path.get('body').getData('emitedCode');
  store i32 %tmp_d, i32* %.i.0.addr         ; i = i + 1;
  %tmp_e = load i32, i32* %.i.0.addr.       ; Code for print(i) emitted during traversal of node.body
  %tmp_f = getelementptr inbounds [4 x i8], [4 x i8]* @.str.i32, i64 0, i64 0
  %tmp_g = call i32 (i8*, ...) @printf(i8* %tmp_f, i32 %tmp_e) ; print(i);
  br label %while.cond_0                    ; `  br label %${condLabel}`. The label was obtained with const condLabel = path.getData('condLabel') at exit of the WhileStatement visitor.
while.end_2:                                ; `${endLabel}:`
  ret i32 0
}
```  

## Translating a WhileStatement from Dragon to LLVM IR

We use `path.setData` to store emitted code for different parts of a `WhileStatement` during LLVM IR code generation, and then retrieving that data later to build the final code for the while loop.

```js
        // ...
        WhileStatement: {
            enter(path) {
                const startIdx = ctx.emitedCode.length;
                path.setData('startIdx', startIdx);

                const condLabel = ctx.nextLabel('while.cond'); // while.cond_0: Generate a unique label for the condition block
                const bodyLabel = ctx.nextLabel('while.body'); // while.body_1: Generate a unique label for the body block
                const endLabel = ctx.nextLabel('while.end');   // while.end_2: Generate a unique label for the end block

                path.setData('condLabel', condLabel); // Store the condition label in the path's data
                path.setData('bodyLabel', bodyLabel); // Store the body label in the path's data
                path.setData('endLabel', endLabel);   // Store the end label in the path's data
            },
            exit(path) {
                visitWhileStatement(path, ctx, nodeValues, { dragonTypeToLLVM });
            }
        },
        //...
```

## visitWhileStatement 

We also use 
`path.get("body")`,
`path.get("test")`
to get a NodePath for the `body` and  `test` nodes so that we can then use 
`path.get("body").getData('startIdx')`, 
`path.get("test").getData('startIdx')` 
to find the index in the emitted code array where the code for the body starts.  
We repeat the same pattern to get the emitted code for the condition and the body: 
`path.get('test').getData('emitedCode')` 
and 
`path.get('body').getData('emitedCode')` 
respectively. See section [docs/babel-traverse/path-navigation.md](/docs/babel-traverse/path-navigation.md) for more details.


```js
/**
 * Handles WhileStatement nodes for LLVM IR codegen.
 * Emits control flow with conditional and unconditional branches.
 */
function visitWhileStatement(path, ctx, nodeValues) {
    const node = path.node;
    // Retrieve labels and startIdx from the path
    const condLabel = path.getData('condLabel'); // Get the condition label from the path's data
    const bodyLabel = path.getData('bodyLabel'); // Get the body label from the path's data
    const endLabel = path.getData('endLabel');   // Get the end label from the path's data
    const startIdx = path.getData('startIdx');   // Get the start index for emitted code from the path's data
    const bodyStartIdx = path.get("body").getData('startIdx'); // <-- Get startIdx for the body to slice emitted code later

    const codeBeforeWhile = ctx.emitedCode.slice(0, startIdx);
    
    // After traverse, code for cond+body is now in emitedCode[startIdx:]
    // Find split point: all code for node.test comes first, then node.body
    const condVal = getNodeValue(nodeValues, node.test);

    if (!condVal) return; // Unlikely, but must be due to JS AST mismatch?

    const condCode = path.get('test').getData('emitedCode'); // Get emitted code for the condition expression
    const bodyCode = path.get('body').getData('emitedCode'); // Get emitted code for the body statement(s)

    /* Rebuild code buffer!  */
    const whileCode = buildCode(
      `${condLabel}:`,
         condCode,
      `  br i1 ${condVal.value}, label %${bodyLabel}, label %${endLabel}`,
      `${bodyLabel}:`,
          bodyCode,
       `  br label %${condLabel}`,
      `${endLabel}:`
    )
    // If the code before the while loop did not emit a branch to the condition, add it now to make it a correct basic block. 
    if (!lastEmittedIsBranchOrReturn(codeBeforeWhile)) {
        whileCode.unshift(`  br label %${condLabel}`);
    }
    
    ctx.emitedCode = buildCode(codeBeforeWhile, whileCode);

    // Save the code for this node in the path
    path.setData('emitedCode', whileCode);
}
```

See [src/llvm/visitor-helpers.cjs](/src/llvm/visitor-helpers.cjs#L11-L15) for the helper function `lastEmittedIsBranchOrReturn` used to check if the last emitted instruction is a control flow terminator.

## Functions and Basic Blocks
In LLVM IR, control flow is structured around **basic blocks** and **labels**. A basic block is a straight-line code sequence with no branches except at the end. Each basic block starts with a label (e.g., `while.cond_0:`) that serves as a target for branch instructions.

Here is an excerpt from the section [Functions](https://llvm.org/docs/LangRef.html#functions) if the LLVM Language Reference Manual:

> <p>A function definition contains a list of basic blocks, forming the CFG (Control Flow Graph) for the function. Each basic block may optionally start with a label (giving the basic block a symbol table entry), contains a list of instructions and <a class="reference internal" href="https://llvm.org/docs/LangRef.html#debugrecords"><span class="std std-ref">debug records</span></a>, and <b>ends with a <a class="reference internal" href="https://llvm.org/docs/LangRef.html#terminators"><span class="std std-ref">terminator</span></a> instruction (such as a branch or function return</b>). If an explicit label name is not provided, a block is assigned an implicit numbered label, using the next value from the same counter as used for unnamed temporaries (<a class="reference internal" href="https://llvm.org/docs/LangRef.html#identifiers"><span class="std std-ref">see above</span></a>). For example, if a function entry block does not have an explicit label, it will be assigned label ”%0”, then the first unnamed temporary in that block will be “%1”, etc. If a numeric label is explicitly specified, it must match the numeric label that would be used implicitly.</p>

> <p>The first basic block in a function is special in two ways: it is immediately executed on entrance to the function, and it is not allowed to have predecessor basic blocks (i.e., there can not be any branches to the entry block of a function). Because the block can have no predecessors, it also cannot have any <a class="reference internal" href="https://llvm.org/docs/LangRef.html#i-phi"><span class="std std-ref">PHI nodes</span></a>.</p>
